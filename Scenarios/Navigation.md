# Navigation and layout

## Nav tabs – signed-in user

If the user is signed in, then the main navigation shows links such as Dashboard, Bookings, Actions, Plugin, and More. Clicking a link takes the user to that page. The link for the current page may be highlighted. If the user has a company (business ID), a "Booking form" (or equivalent) link may appear for designing the booking form.

## Super Admin – Manage menu

If the signed-in user is a Super Admin, then the navigation shows a "Manage" menu (or dropdown). Opening it shows links to Users and Companies. Clicking one goes to the manage users or manage companies page.

## Account and sign out

If the user is signed in, then the navigation shows an account control (for example the user's name or "Account"). Clicking it opens a dropdown with a "Sign out" option. If the user clicks "Sign out", then the app clears the session and redirects the user to the login page.

## Not signed in

If the user is not signed in, then they see the login (or landing) page. They can go to Register or Forgot password from there. They do not see the main app navigation until they sign in.

## Locale and theme

If the app supports multiple languages, then the user can switch locale (for example EN, FI) from the navigation. If the app supports theme (light/dark), then the user can switch theme. These actions update the UI without changing the current page.

# UI: Multi-language (i18n)

## Approach

- **next-intl**: Middleware for locale (e.g. `fi`, `en`); `[locale]` dynamic segment for all routes; load `messages/[locale].json` and pass to `NextIntlClientProvider` in layout.
- **Usage**: `useTranslations('namespace')` in components; structure messages by feature (e.g. `auth`, `bookings`, `common`) in each JSON file.
- **Locales**: At least **fi** and **en**; document how to add a new locale (new JSON file, add locale to config and middleware).

## Switcher

- Locale switcher in navbar or user menu: switch to `/fi/...` or `/en/...` (and set cookie if using next-intl’s default locale persistence).

## Adding a locale

- Add `messages/<locale>.json` with same keys as `fi.json`/`en.json`.
- Add locale to next-intl config and middleware `locales` array.
- No code changes in components if keys are shared.
